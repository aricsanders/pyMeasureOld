The directory structure in pyMeasure is :

pyMeasure

Top folder with the main application, the file register and metadata for each file.

Instruments

1st level folder that contains information regarding instruments in xml format

Experiments

1st level folder that contains information regarding experiments in xml format

Data

Measurements

Logs

States

Reports

Settings

Persistent Data for Use by the application (images, preferences etc.)

Code

FrontEnds- Code that deals with user interfaces (uses wx)

BackEnds- Code that talks with instruments (uses visa)

DataHandlers- Code that stores and processes data (uses xml)

Misc- If it does not involve a user interface, talking with an instrument, or processing data


Code

I have tried to keep the code as modular and useful as possible. I have intentionally separated out things that talk to instruments, user interfaces, and data manipulation so that the code could used elsewhere.

Where possible I have tried to use robust programming styles and tried to be consistent in my naming and documenting.


FrontEnds

I have used the Boa Constructor RAD platform for a lot of the user interfaces. It is a program that allows quicker design of GUI's using the wxWindows library. To be truly modular

  1. Define Guts in a wx.panel

  2. Define Several Frames with tool bars,etc

  3. Using the Custom classes in BOA Combine

In addition, using the other packages in pyMeasure I have tried to make the user interfaces a little more readable by only calling functions from elsewhere.
